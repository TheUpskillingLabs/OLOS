Identical code; only the banner changes. It reads the deployment's Supabase URL, so an organizer always knows whether they're looking at real participant data. No per-environment logic anywhere else.

The data-vs-filters question, settled for v1: we never dump a full table — every view is server-paginated (50/page). Three structural filters are always on: the entity selector, the cycle filter (applied automatically to cycle-scoped entities), and the show-deleted toggle. A free-text / per-column search is intentionally v2 — the disabled box on the right shows where it lands. If we find we need rich filtering, that's usually the signal we're ready to build the real admin instead of extending this stopgap.
